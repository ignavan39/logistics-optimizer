# Pitfalls — Антипаттерны (НЕ делать)

> "Грабли" — ошибки и антипаттерны которые встречались в проекте.

---

## Backend Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|------------|--------------|------------|
| `process.env` напрямую | Ломается в Docker | `ConfigService.get()` |
| `synchronize: true` в продакшене | Миграции в проде | `synchronize: NODE_ENV === 'development'` |
| Kafka без идемпотентности | Дубликаты событий | Проверяй `eventId` перед обработкой |
| `@ts-ignore` | Скрывает ошибки | Типизируй правильно |
| Seed в `apps/` | Тестовые данные в коде | `infra/postgres/seeds/*.sql` |

---

## Frontend Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|------------|--------------|------------|
| `useEffect` для фетчинга | Race conditions | React Query (`useQuery`) |
| >50 маркеров без кластеризации | Лагает карта | MarkerCluster или виртуализация |
| `any` в TypeScript | Теряется типизация | Всегда типизируй |
| Hardcoded URL | Не работает в проде | ENV variables |

---

## Как добавлять Pitfalls

Когда находишь грабли:

1. Запиши что НЕ делать
2. Объясни почему плохо  
3. Дай правильную альтернативу
4. Добавь пример кода

---

## Пример добавления

```markdown
## Название антипаттерна

**Плохо:**
\`\`\`typescript
const value = any
\`\`\`

**Почему:** Теряется безопасность типов.

**Правильно:**
\`\`\`typescript
const value: string = '明确'
\`\`\`
```
