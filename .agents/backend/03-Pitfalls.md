# Pitfalls — Антипаттерны (НЕ делать)

> "Грабли" — ошибки и антипаттерны которые встречались в проекте.

---

## 🎯 Главный принцип

**Эволюционируй вместе с проектом.** Нашёл новый антипаттерн → запиши сюда.

---

## Backend Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|-------------|-------------|----------|
| `process.env` напрямую | Ломается в Docker, не тестируемо | `ConfigService.get()` |
| `synchronize: true` в production | Миграции в проде могут дропнуть данные | `synchronize: NODE_ENV === 'development'` |
| Kafka без идемпотентности | Дубликаты при rebalance/retry | Проверяй `eventId` перед обработкой |
| `@ts-ignore` | Скрывает ошибки типов | Типизируй правильно |
| Seed в `apps/` | Тестовые данные в коде | `infra/postgres/seeds/*.sql` |
| `@nestjs/typeorm` в Docker | Конфликт с pnpm hoisting | `DataSource` напрямую |
| Одинаковые порты test + prod | Конфликт при docker compose | ports 6400+ для test (6433, 6434...) |
| Полагаться на init SQL | docker-entrypoint-initdb.d выполняется только при ПЕРВОМ запуске | Проверяй схему вручную |
| gRPC readiness = true | waitForReady может быть true, но методы возвращают Internal (неполная схема) | Тестируй реальные методы |

## Frontend Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|-------------|-------------|----------|
| `useEffect` для фетчинга | Race conditions | React Query (`useQuery`) |
| >50 маркеров без кластеризации | Лагает карта | MarkerCluster или виртуализация |
| `any` в TypeScript | Теряется типизация | Всегда типизируй |
| Hardcoded URL | Не работает в проде | ENV variables |

---

## Git Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|-------------|-------------|----------|
| `git checkout -- .` | Откатывает ВСЕ изменения, включая неотслеживаемые | `git restore` для отдельных файлов |

---

## Architecture Anti-patterns

| Антипаттерн | Почему плохо | Правильно |
|-------------|-------------|----------|
| Микросервис "на будущее" | Излишняя сложность, network overhead, distributed transactions | Начинай с модульного монолита, выделяй сервис только при явных признаках |

## Как добавлять Pitfalls

```markdown
## Название антипаттерна

**Плохо:**
\`\`\`typescript
const value = any
\`\`\`

**Почему:** Теряется безопасность типов.

**Правильно:**
\`\`\`typescript
const value: string = ''
\`\`\`
```

## Важно!

1. **init SQL работает только при первом запуске контейнера** — симптом: "column does not exist" после перезапуска
2. **test ports 6400+** — избегай конфликта с production (5433)
3. **gRPC не только readiness** — тестируй реальные методы
